VScanMail — FRONTEND MAP FOR BACKEND DEVELOPERS
================================================================================
Generated as a route and integration reference. Frontend-only; no backend
implementation details. Paths are URL paths (Next.js App Router).

--------------------------------------------------------------------------------
1. STACK & CONFIG
--------------------------------------------------------------------------------
  Framework     : Next.js 16 (App Router)
  UI            : React 19, Tailwind CSS 4
  Package root  : frontend/
  Dev server    : npm run dev  (default http://localhost:3000)

  Backend base URL (client-side):
    Env var       : NEXT_PUBLIC_API_URL
    Example       : http://localhost:5000
    Fallback      : http://localhost:5000  (if env unset)
    See           : frontend/.env.local.example

  No Next.js middleware.ts in this repo (auth/routing guards not centralized
  here at time of writing).

--------------------------------------------------------------------------------
2. API ROUTES THE FRONTEND ALREADY CALLS
--------------------------------------------------------------------------------
  Base: {NEXT_PUBLIC_API_URL} with no trailing slash.

  Customer account
    GET  /api/customer/account?companyId={id}
    PUT  /api/customer/account?companyId={id}
         Body: JSON partial { profile, bankAccounts, security, notifications }
    Source: frontend/src/lib/customerAccount.ts

  Customer billing
    GET  /api/customer/billing?companyId={id}
    POST /api/customer/billing/upgrade-request
         Body: JSON { companyId, planId }
    Source: frontend/src/lib/customerBilling.ts

  Note: Many screens still use static mock data (frontend/src/mocks/*.ts) or
  inline arrays; those areas will need matching backend endpoints when wired.

8. NOTES FOR BACKEND ALIGNMENT
--------------------------------------------------------------------------------
  • companyId appears in existing customer API calls (often hardcoded as
    "demo" in UI until auth provides a real id).
  • Admin/superadmin list screens expect row shapes aligned with mocks until
    replaced by real fetches.
  • External image URLs: some pages use third-party image APIs (e.g. readdy.ai)
    for placeholders; unrelated to your REST API.
